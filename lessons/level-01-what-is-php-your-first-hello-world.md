# Level 1: What is PHP? Your first 'Hello World'

**Track:** PHP · **Level:** 1/12 · **Difficulty:** `beginner`

📚 **Today's lesson** — published 2026-06-27

## TL;DR

PHP is a server-side scripting language that runs on the web server and sends HTML to the browser. The browser never sees PHP code — only the output PHP produces.

## Real-world analogies

- PHP is like a chef in a restaurant kitchen. You (the browser) place an order (request). The chef (PHP) cooks the dish (HTML) and sends it out. You never see the chef's recipe book (the .php file).
- HTML is the finished sandwich. PHP is the kitchen that makes the sandwich.

## Key concepts

### `Server-side`

Code that runs on the web server, not in the browser. The browser only sees the final HTML.

### `Interpreter`

PHP is read line-by-line each time someone visits the page. Unlike compiled languages (C, Go), there's no build step.

### `File extension`

Files end in `.php` so the server knows to run them through PHP. `.html` files skip PHP entirely.

### `Tags`

PHP code lives between `<?php` and `?>` tags. Everything outside those tags is sent to the browser as-is.

## Code with comments

Every line has a comment. Read it slowly.

```php
<?php
// This is a single-line comment. PHP ignores it.

/*
This is a multi-line comment.
Useful for explaining tricky code.
*/

// echo is PHP's way of saying "send this text to the browser"
echo "Hello, world!";

// You can mix HTML with PHP
?>
<!DOCTYPE html>
<html>
<head><title>My first PHP page</title></head>
<body>
    <h1><?php echo "PHP can run inside HTML like this."; ?></h1>
    <p>The current time is: <?php echo date('h:i:s A'); ?></p>
</body>
</html>
<?php
// You can switch back to PHP mode any time
echo "<!-- This comment is hidden in the page source -->';
```

## Try it yourself

Create a file called `hello.php`. Inside, print your name and the current date. Open it in your browser (after starting a local server with `php -S localhost:8000`).

## Common pitfalls

- ⚠️ PHP must be installed on your computer. Download from php.net or use XAMPP/MAMP for an easy all-in-one.
- ⚠️ You can't run a `.php` file by double-clicking it. You need a web server (PHP's built-in one is fine for learning).
- ⚠️ If you see the raw `<?php ... ?>` text in your browser, the server isn't running PHP — it's serving the file as plain text.
