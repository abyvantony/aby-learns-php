# Level 7: String manipulation and regex

**Track:** PHP · **Level:** 7/12 · **Difficulty:** `intermediate`

📚 **Today's lesson** — published 2026-06-29

## TL;DR

Strings are everywhere — names, emails, JSON, HTML. PHP has dozens of string functions. Learn the most-used ones, then reach for regex when you need pattern matching.

## Real-world analogies

- String functions are a Swiss Army knife for text: `strlen` is the ruler, `strtoupper` is the loud voice, `str_replace` is find-and-replace.
- Regex is a powerful but cryptic pattern language — like a search query on steroids. `\d+` means 'one or more digits'.

## Key concepts

### ``strlen` and `mb_strlen``

`strlen` counts bytes. `mb_strlen` counts characters (matters for emoji and non-ASCII text).

### ``strpos` vs `str_contains``

PHP 8 added `str_contains($haystack, $needle)` — finally, a clear name. Before that, you used `strpos !== false`.

### ``sprintf``

Printf-style formatting. `sprintf('Hi %s, you are %d', 'Aby', 30)` — great for building consistent output.

### `Heredoc / Nowdoc`

Multi-line strings. Heredoc (`<<<EOT`) interpolates variables; nowdoc (`<<<'EOT'`) doesn't.

## Code with comments

Every line has a comment. Read it slowly.

```php
<?php
$text = "  Hello, World!  ";

strlen($text);                // 16
trim($text);                  // "Hello, World!"  (removes whitespace)
strtolower("HELLO");          // "hello"
strtoupper("hello");          // "HELLO"
ucfirst("hello world");       // "Hello world"
ucwords("hello world");       // "Hello World"

str_contains("Hello", "ell");  // true (PHP 8+)
str_starts_with("Hello", "He"); // true (PHP 8+)
str_ends_with("Hello", "lo");   // true (PHP 8+)
str_replace("World", "PHP", $text);  // "  Hello, PHP!  "

substr("Hello World", 0, 5);  // "Hello" (offset 0, length 5)
substr("Hello World", 6);     // "World" (offset 6, rest)
substr("Hello World", -5);    // "World" (last 5 chars)

explode(",", "a,b,c");         // ["a", "b", "c"]
implode("-", ["a", "b", "c"]); // "a-b-c"

// sprintf — printf-style
sprintf("Name: %s, Age: %d", "Aby", 30);  // "Name: Aby, Age: 30"
printf("Pi is approximately %.2f\n", 3.14159);  // "Pi is approximately 3.14"

// Heredoc — multi-line, interpolates
$name = "Aby";
$message = <<<EOT
Hello, $name!
This is a multi-line string.
Variables like $name get replaced.
EOT;

// Nowdoc — multi-line, no interpolation
$code = <<<'EOT'
<?php
echo "Hello, \$name";  // $name is literal, not replaced
EOT;

// Regular expressions (preg_*)
$email = "aby@example.com";
preg_match("/^[\w.+-]+@[\w.-]+\.\w+$/", $email);  // 1 if valid

// preg_match_all — find all matches
$text = "Call 555-1234 or 555-5678";
preg_match_all("/\d{3}-\d{4}/", $text, $matches);
// $matches[0] = ["555-1234", "555-5678"]

// preg_replace — find and replace
$slug = strtolower("Hello World!");
$slug = preg_replace("/[^a-z0-9]+/", "-", $slug);
$slug = trim($slug, "-");  // "hello-world"
```

## Try it yourself

Write a function that takes an email and returns true if it's valid. Use `filter_var` first, then write your own using regex. Compare both approaches.

## Common pitfalls

- ⚠️ `strpos` returns `false` (not -1) on miss and `0` (a valid index) on hit. Use `!== false` for comparisons.
- ⚠️ `str_replace` with arrays replaces all occurrences: `str_replace(['a', 'b'], ['x', 'y'], 'abc')` → 'xyc'.
- ⚠️ Regex character classes are picky: `\d` is digits, `\w` is word chars (letters, digits, underscore), `.` matches anything but newline.
