# Level 9: Object-Oriented PHP: classes, objects, methods

**Track:** PHP · **Level:** 9/12 · **Difficulty:** `intermediate-plus`

📚 **Today's lesson** — published 2026-06-30

## TL;DR

OOP lets you bundle data and behavior into reusable 'objects'. A class is the blueprint, an object is the thing built from it. PHP's OOP is mature and similar to Java/C#.

## Real-world analogies

- A class is a cookie cutter. An object is a cookie. The cutter defines the shape, but each cookie is a separate, edible thing.
- Methods are actions the object can do. Properties are facts about the object. Together they make the object 'alive' in your code.

## Key concepts

### ``$this``

Inside a class method, `$this` refers to the current object. `$this->name` means 'the `name` property of THIS particular object'.

### `Constructor`

`__construct()` runs when you create a new object. Use it to set up initial state.

### `Visibility`

`public` (anyone can access), `protected` (this class + subclasses), `private` (this class only). Default is `var`/public.

### `Static`

`static` properties/methods belong to the class itself, not to any object. Access with `ClassName::$prop` or `ClassName::method()`.

## Code with comments

Every line has a comment. Read it slowly.

```php
<?php
class User {
    // Properties
    public string $name;
    private int $age;
    protected array $roles = [];

    // Constructor — runs on `new User(...)`
    public function __construct(string $name, int $age) {
        $this->name = $name;
        $this->age = $age;
    }

    // Method
    public function introduce(): string {
        return "Hi, I'm {$this->name}.";
    }

    // Getter for private property
    public function getAge(): int {
        return $this->age;
    }

    // Static method — called on the class, not an instance
    public static function createGuest(): self {
        return new self("Guest", 0);
    }
}

$aby = new User("Aby", 30);
$aby->introduce();          // "Hi, I'm Aby."
$aby->name;                 // "Aby" (public)
$aby->getAge();             // 30 (via getter)
$guest = User::createGuest();

// Inheritance
class AdminUser extends User {
    public function banUser(User $user): void {
        echo "{$this->name} banned {$user->name}";
    }
}

$admin = new AdminUser("Boss", 40);
$admin->banUser($aby);      // "Boss banned Aby"

// Abstract class — can't be instantiated directly
abstract class Shape {
    abstract public function area(): float;
}

class Circle extends Shape {
    public function __construct(private float $radius) {}

    public function area(): float {
        return pi() * $this->radius ** 2;
    }
}

// Interface — contract that classes must fulfill
interface Loggable {
    public function log(string $message): void;
}

class FileLogger implements Loggable {
    public function log(string $message): void {
        file_put_contents("app.log", "[$message]\n", FILE_APPEND);
    }
}

// Traits — reusable code across classes (PHP's answer to multiple inheritance)
trait Timestampable {
    public function createdAt(): string {
        return date('Y-m-d H:i:s');
    }
}

class Article {
    use Timestampable;
    public function __construct(public string $title) {}
}

$article = new Article("Hello");
$article->createdAt();  // "2026-06-27 14:30:00"

// Readonly properties (PHP 8.1+)
class Point {
    public function __construct(
        public readonly float $x,
        public readonly float $y
    ) {}
}
$p = new Point(1.0, 2.0);
$p->x = 5;  // Error! readonly
```

## Common pitfalls

- ⚠️ Forgetting `private` makes properties `public` by default — anyone can change them. Use accessors for sensitive data.
- ⚠️ `==` on objects compares properties. `===` checks identity (same object in memory).
- ⚠️ Static methods can't use `$this` — they have no object context. They belong to the class itself.
