# Level 2: Variables, types, and printing

**Track:** PHP · **Level:** 2/12 · **Difficulty:** `beginner`

📚 **Today's lesson** — published 2026-06-27

## TL;DR

Variables are named boxes that hold a value. PHP has 8 basic types but you'll mainly use strings, integers, floats, booleans, arrays, and null. PHP figures out the type automatically — you don't declare it.

## Real-world analogies

- A variable is a labeled jar. The label is the variable name, what's inside is the value. You can empty the jar and put something else in it any time.
- PHP's loose typing is like a magic wand that figures out what kind of thing you put in the jar. `42` becomes a number, `"42"` becomes text.

## Key concepts

### ``$` prefix`

Every variable name starts with `$`. `$name`, `$age`, `$price`. The dollar sign is PHP's way of saying 'this is a variable'.

### `Assignment`

`=` doesn't mean 'equals' in math. It means 'put the right-side value into the left-side variable'.

### `String interpolation`

Double-quoted strings let you embed variables: `"Hello, $name"`. Single-quoted strings don't: `'Hello, $name'` prints literally.

### `Type juggling`

PHP converts types automatically in many situations. `"5" + 3` becomes `8` (the string becomes a number). This is convenient but causes bugs.

## Code with comments

Every line has a comment. Read it slowly.

```php
<?php
// Declaring variables — no 'var' or 'let' keyword needed
$name = "Aby";           // string (text)
$age = 30;               // integer (whole number)
$height = 1.75;          // float (decimal)
$isLawyer = true;        // boolean (yes/no)
$nothing = null;         // null (intentionally empty)

// Single quotes = literal, no variable substitution
$literal = 'My name is $name';        // My name is $name
// Double quotes = variables get inserted
$interpolated = "My name is $name";   // My name is Aby
$withBraces = "I am {$age} years old"; // I am 30 years old

// Printing
echo $name;                          // Aby
echo "Hello, $name!";                // Hello, Aby!
echo 'I am ' . $age . ' years old';  // I am 30 years old (concatenation with .)

// var_dump shows the type AND value — invaluable for debugging
var_dump($name);    // string(3) "Aby"
var_dump($age);     // int(30)
var_dump($height);  // float(1.75)

// gettype returns just the type as a string
echo gettype($isLawyer);  // "boolean"

// Type juggling example
$result = "5" + 3;     // PHP converts "5" to 5, result is 8 (int)
$result2 = "5 cats" + 3; // PHP converts "5 cats" to 5, result is 8 (int)
                        // (PHP reads digits from the start of the string)
```

## Try it yourself

Declare variables for your name, age, favorite color, and whether you like PHP. Print a sentence using string interpolation. Use `var_dump` to inspect each variable's type.

## Common pitfalls

- ⚠️ Forgetting the `$` makes PHP think you're referring to a constant.
- ⚠️ Confusing `==` (comparison, with type juggling) vs `===` (strict comparison). `"5" == 5` is `true`, but `"5" === 5` is `false`.
- ⚠️ Single quotes don't interpolate — use double quotes when you need to embed variables.
